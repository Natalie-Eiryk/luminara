## Validation Methodology

### Three-Tier Validation System

**Tier 1: Technical Compliance** (Automated)
- File format (PNG, JPEG acceptable)
- Resolution (1024x1024 minimum for high-detail styles)
- Color depth (8-bit RGB minimum, 16-bit for photorealistic)
- File integrity (non-corrupted, loadable)

**Tier 2: Multi-Stakeholder Assessment** (Automated via Quality Studio)
- Teachers Panel: Educational clarity >= 0.85
- NVIDIA Panel: Technical quality >= 0.90 (varies by style)
- Renaissance Panel: Artistic merit >= 0.85
- Consensus threshold: Average >= 0.85

**Tier 3: Visual Element Checklist** (Manual + Automated)
- Style-specific characteristics (see per-style checklists below)
- Cross-style distinctiveness (no contamination)
- Subject recognizability (red fox identifiable)
- Educational value (demonstrates principles clearly)

## Validation Workflow

### Automated Validation (Tier 1 + Tier 2)

**Script**: `validate_exemplar.py` (TO BE CREATED)

```python
def validate_exemplar(image_path: str, style_name: str) -> ValidationReport:
    """
    Automated validation combining technical checks and quality assessment.

    Args:
        image_path: Path to exemplar PNG
        style_name: One of [photorealistic, impressionism, art_nouveau, pixel_art, ukiyo_e]

    Returns:
        ValidationReport with pass/fail status and detailed metrics
    """
    report = ValidationReport()

    # Tier 1: Technical compliance
    report.technical = check_file_integrity(image_path)
    report.technical.resolution = check_resolution(image_path, min_size=1024)
    report.technical.color_depth = check_color_depth(image_path)

    # Tier 2: Multi-stakeholder assessment
    report.quality = QualityStudioBridge.evaluate(image_path)
    report.quality.teachers = report.quality.teachers_score >= 0.85
    report.quality.nvidia = report.quality.nvidia_score >= get_style_threshold(style_name)
    report.quality.renaissance = report.quality.renaissance_score >= 0.85

    return report
```

## Rejection Criteria and Refinement

### Automatic Rejection Triggers

An image is **REJECTED** if:
- Tier 1 fails (corrupted file, wrong format, insufficient resolution)
- Consensus score < 0.70 (>15% below threshold)
- <40% of visual elements present (major style violation)
- Subject unrecognizable (red fox not identifiable)
- Cross-contamination detected (multiple styles present)

### Refinement Workflow

**When image rejected**:
1. Analyze failure modes (which tier/checklist items failed)
2. Adjust ComfyUI parameters:
   - Increase CFG scale for style adherence
   - Adjust negative prompt to suppress contamination
   - Change sampler if artifacts present
   - Increase steps for detail styles
3. Regenerate using Quality Studio refinement loop (max 50 iterations)
4. Re-validate with same checklist
5. If fails again: escalate to manual prompt revision

**Maximum Refinement Attempts**: 3 per style before prompt revision required
